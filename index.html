Three commands

Everything you need.
Nothing you don't.

Each command is a focused tool. Together they cover the three things every Claude Code user manages manually: skills, memory, and security.

/forgein optimize

Discover skills
before you need them.

Reads your CLAUDE.md, memory files, and git log. Scores the entire Forgein registry against your actual workflow — not what you think you need, but what you provably do. Recommends and installs in one step.

  • Word-boundary signal matching — "pr" won't match "sprint"
  • Excludes already-installed skills automatically
  • Default is install-all — no hunting for the right input
  • Fetches skill files directly from GitHub via gh API

/forgein mem

Your memory,
under control.

Claude Code's auto-memory system builds context over time — but it's write-only from your side. Forgein mem gives you full CRUD: list what's saved, search it, add entries, prune stale ones, and audit the index for structural issues.

  • Groups memories by type: user · feedback · project · reference
  • Prune verifies referenced PRs, files, and functions still exist
  • Audit finds broken links, duplicates, and missing frontmatter
  • Works across all 15+ project memory directories

/forgein sec

Security check
before every push.

Runs a heuristic security scan against staged git changes — or any file or directory you point it at. Checks eight vulnerability classes and ranks findings by severity. Honest about what it is: a vibe check, not a full audit.

  • Secrets, SQL/command injection, path traversal, SSRF, XSS
  • Insecure deserialization and hardcoded credentials
  • If no staged changes, asks for a path — never scans silently
  • Critical → high → medium ranking with attack scenarios

How optimize works

Signal matching,
not guessing.

optimize doesn't ask you what you need. It reads your actual workflow context and scores every skill in the registry against it — whole-word matching, so "pr" won't fire on "sprint".

STEP 01
Fetch registry
Pulls registry.json via gh api — no CDN lag, no auth issues. Falls back to raw GitHub if gh isn't available.
STEP 02
Read context
Reads CLAUDE.md (global + project), memory index files, and git log. Builds a single context string from all sources.
STEP 03
Score skills
For each registry skill: counts whole-word matches of its signals and tags in your context. Excludes already-installed skills.
STEP 04
Install
Presents top 5 with matched signals. Default is Enter = install all. Fetches each skill file and writes to ~/.claude/commands/.
# Scoring formula score(skill) = Σ whole_word_match(term, context) for term in skill.signals + skill.tags   # Whole-word match: char before/after must be non-alphanumeric # "pr" → matches "open prs" ✓ "pr" → does NOT match "sprint" ✗   # Context sources (all concatenated) context = read(~/.claude/CLAUDE.md) + read(./CLAUDE.md) + find(MEMORY.md files) + git log --oneline -20
